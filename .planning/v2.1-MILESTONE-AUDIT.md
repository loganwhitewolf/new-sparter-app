## E2E Flows

### Flow 1: Import → Auto-Discovery → CTA → Suggestions → Promote → Inline Count

1. `confirmImportAction` → `importFile` → categorization pipeline → post-commit `discoverRegexCandidates` → `discoveryCount`
2. `import-preview.tsx` receives `discoveryCount > 0` → CTA with link to `/import/{fileId}/suggestions`
3. RSC: `verifySession` + `getFileForUser` (status=imported guard) + `getPlatformIdForUserFile` + `discoverRegexCandidates`
4. `SuggestionSection` renders two sections (SUMUI-02); `SuggestionCard` per candidate
5. User promotes → `promoteSuggestionAction`: `verifySession` → `getPlatformIdForUserFile` (IDOR) → `createPattern` → `applyNewPatternToPlatformExpenses` → returns `applyResult`
6. `SuggestionPromoteForm` `useEffect` → `onPromoted(state.applyResult)` → `SuggestionCard` inline count: "{N} categorizzate · {M} ancora senza match"

**Status: COMPLETE**

### Flow 2: Files Table → Ricontrolla Regex → Zero Toast or Navigate

1. `handleRecheckRegex` → `recheckRegexAction`: `verifySession` → `getPlatformIdForUserFile` → `discoverRegexCandidates`
2. `total === 0` → `toast("Nessun pattern trovato per questa piattaforma")`, no navigation (D-06 correct)
3. `total > 0` → `router.push(/import/{fileId}/suggestions)`

**Status: COMPLETE**

### Flow 3: Suggestions Page → Promote → Platform-Scoped Apply → Inline Count

Identical to Flow 1 steps 3–6. `applyNewPatternToPlatformExpenses` uses `getUncategorizedExpensesForPlatformApply` (platform-scoped Set B, not user-wide). `PatternApplyResult` travels service → validations re-export → Server Action return → `useActionState` → `SuggestionCard` without type drift.

**Status: COMPLETE**
