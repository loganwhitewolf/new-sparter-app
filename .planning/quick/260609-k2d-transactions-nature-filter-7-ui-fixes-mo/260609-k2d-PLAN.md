- Transactions/Expenses/Files filters are driven by the SHARED `DataTableToolbar` via per-table config files (`app/(app)/<page>/<name>.table.tsx`), NOT by `components/transactions/transaction-filters.tsx` (that legacy Card+Select component is unused — do NOT touch it).
- The nature filter and the in/out/transfer filter are therefore added as `FilterField` entries in `transactions.table.tsx`, parsed in `parseTransactionFilters`, and applied in `getTransactions`. Options are passed via the page's `filterOptions` prop.
- `flowNatureEnum` in `lib/db/schema.ts` has NINE values (essential, discretionary, operational, financial, income, income_extraordinary, debt, extraordinary, transfer). Use `NATURE_ORDER` + `NATURE_LABELS` from `lib/utils/nature-labels.ts` as the canonical option source — do NOT hand-type the CONTEXT.md 7-value list.
- `category.type` enum is `["in","out","system","transfer"]`.
- Import totals (fix 7): `positiveTotal`/`negativeTotal` are already columns on `file` and already on `ImportListRow`. Each is a per-file precomputed total (DECIMAL → string). Fix 7 is render-only formatting; there is NO cross-row aggregation at the list level, so no Decimal math is needed here. (If any incidental money math is added, use `@/lib/utils/decimal` — never native arithmetic.)
