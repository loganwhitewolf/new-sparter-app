# Quick Task 260709-mf6: Restyle KPI cards — recurring-first hierarchy

## Goal (design approved via prototype 2026-07-09)

Make the recurring/structural values stand out — "they explain the trend". Invert the
cards: the **components** become the primary display (stacked rows, recurring emphasized
in full colour), the **grand total** drops below a divider as a summary line.

Approved layout (per stacked prototype):
- **Entrate**: Ricorrenti (2xl, green, dot) / Straordinarie (lg, muted) → divider → Totale
- **Uscite**: Essenziale (2xl, red, dot) / Discrezionale, Debiti (lg, muted) → divider → Totale
- **Bilancio**: Solo ricorrenti (2xl, sign-coloured) → divider → Totale (sign-coloured) + reading
- **Tasso risparmio**: Solo ricorrenti (2xl, sign-coloured) → divider → Totale + reading
- **Accantonato**: single value (2xl, purple) → divider → reading (no total line)

## Decisions / deviations from the mock (flag to user)

- **YoY delta badges kept.** The mock (null deltas) showed none, but the vs-prior-year
  delta is existing data — keep the badge in the footer. Easy to drop later if the
  user prefers the barer look.
- **Trend reading text dropped on Entrate/Uscite** — it duplicated the delta badge; the
  approved mock has no reading there. Bilancio/Tasso/Accantonato keep their readings.
- **Balance/rate now sign-coloured** (green ≥0 / red <0) instead of the old neutral
  `--balance` tone — matches the prototype (structural red, total green).

## Tasks

### 1. Rework `ReadingKpiCard` API (kpi-card-reading.tsx)
- New props: `components: KpiComponentRow[]` (`{ label?, value, tone, emphasis? }`),
  `total?: { value, tone } | null`, `reading?: Reading | null`, plus existing
  `label, delta, prevYear, goodWhenPositive, className`.
- `ValueTone = 'in' | 'out' | 'allocation' | 'muted' | 'neutral'` → colour class map
  (allocation uses arbitrary `[var(--total-allocation)]`; others use registered tokens).
- Emphasised row: value 2xl + tone colour + leading dot; secondary: lg + muted-ish.
- Labelless row (Accantonato) → big value only, left-aligned.
- Footer (when total/reading/delta present): divider, optional "Totale" line, reading,
  delta badge.

### 2. Rewire the five cards (kpi-row.tsx)
- Build `components` + `total` per card from OverviewData (Decimal.js for the derived
  Straordinarie). Null breakdown fields → single-value fallback (classic card:
  grand-total emphasised + trend reading, no total line).
- Sign-based tone for Bilancio/Tasso (structural + total).

### 3. Tests (overview-interactions.test.tsx)
- Rewrite the `ReadingKpiCard` unit tests for the new API (emphasis structure, total
  line, labelless single value, delta badge).
- KpiRow render tests: keep the text assertions (labels/values unchanged); add that
  "Totale" appears and the null-fields fallback still hides breakdown labels.
