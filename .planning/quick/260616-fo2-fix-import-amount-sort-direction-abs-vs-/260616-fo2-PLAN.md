# Quick Task 260616-fo2: Fix import amount sort direction

## Problem

Transaction table displays amounts via `formatAbsoluteAmount` but `buildTransactionOrderBy` sorted on signed `transaction.amount`. Result:

- ↓ DESC: highest **positive** values first (+500 before -1000)
- ↑ ASC: highest **negative** magnitudes first (-500 before +100)

Opposite of user expectation (largest displayed amounts first on ↓).

## Fix

Sort transaction `amount` by `ABS(amount::numeric)` — aligned with D-20 filter semantics and display.
