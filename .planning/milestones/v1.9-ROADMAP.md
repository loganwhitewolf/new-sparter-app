# Milestone v1.9: Social Auth

**Status:** ✅ SHIPPED 2026-05-22
**Phases:** 30–32
**Total Plans:** 9

## Overview

Enabled Google and GitHub OAuth authentication for Sparter. Users can sign in or register with social providers, and existing email/password account holders can link or unlink providers from their settings page. The registration guardrail (ALLOWED_EMAIL check) was removed to allow free signup via OAuth. The settings navigation was reorganized with a hub at /settings and a dedicated /settings/profile page hosting the new ConnectedAccountsCard.

## Milestone Summary

**Key Decisions:**

- Guard OAuth provider activation on CLIENT_ID only (not CLIENT_SECRET) — presence of ID is the activation signal; missing SECRET crashes loudly at first OAuth attempt (D-03/D-04)
- OAuth vars remain commented out in `.env.example` to prevent accidental secret commit by copy-paste
- Callback URL pattern: `{BETTER_AUTH_URL}/api/auth/callback/{provider-id}` — documented generically in runbook
- `SocialProviderButtons.errorCallbackURL` is overridable per-page so /login and /register can each own their error URL (D-07)
- `lucide-react@1.14.0` has no Github icon — replaced with inline SVG `GithubIcon` component (Rule 1 bug fix)
- `canUnlink` guard checks credential OR other social (more robust than total-account-count check)
- `configuredProviders` derived from `process.env` booleans in the server component — no `NEXT_PUBLIC_*` vars introduced
- `errorCallbackURL` passed bare so Better Auth appends the real error code as a query param (D-12 fix)
- `/settings` is a hub (no redirect); `/profile` is a pure redirect shim to `/settings/profile` for backward compat

**Issues Resolved:**

- Registration guardrail removed — previously blocked non-approved emails from registering; now any OAuth account can register freely
- React 19 `renderToStaticMarkup` encodes apostrophes as `&#x27;` — Wave 0 test assertion updated accordingly

**Issues Deferred:**

- LINK-01..04 live OAuth round-trip tests are `test.fixme()` stubs — require real OAuth credentials configured for dev URL
- R038/R039/R041 — live Vercel/Supabase/R2 deploy remains operator-pending
- R029 — categorization revalidation partial (carry to next milestone)

**Technical Debt Incurred:**

- None significant — implementation followed plan exactly
