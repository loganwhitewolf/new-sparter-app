<objective>
Build the `TagFilterSelect` client control (the visible UI for TAG-04's global tag filter) and
add the third "Tag" tab to `DashboardTabNav`, extending `buildDashboardTabHref` to carry `?tag=`
across tab switches the same way `preset`/`type`/`sort` already are.

Purpose: This is the only piece of TAG-04 that is pure UI — everything else in this phase is
either a DAL predicate (Plan 68-01/02/03) or page-level wiring that consumes this component
(Plan 68-06). Building it standalone here, against real `TagRow[]` props, keeps it independently
testable and unblocks Plan 68-06 without waiting on the DAL threading plans.

Output: A reusable `TagFilterSelect` component and a 3-tab `DashboardTabNav` that both correctly
carry `?tag=`.
</objective>
