<objective>
Implement D-08's "smart back" in the one shared place all three detail pages consume:
`DetailPageShell`. Today the shell's back link is a plain `next/link` `<Link href={backHref}>`
— clicking it always navigates to the static table route, discarding whatever ephemeral
filters/sort/scroll position the user had when they clicked into the row. This plan makes it
try `router.back()` first (preserving history-based state) and only fall back to the static
`backHref` route when there is no usable in-app history (direct visit, bookmark, or arriving
from another detail page's cross-reference link).

Purpose: this is the single missing piece of "breadcrumb/back behavior is consistent" (DET-09,
64-CONTEXT.md D-07/D-08). Because all three detail pages already share `DetailPageShell`,
fixing it once here retroactively completes the "consistent back" requirement for the
transaction and expense pages shipped in Phase 63, not just the new file page from this phase.

Output: `DetailPageShell`'s back affordance becomes a client-side smart-back control; its
existing `backHref` prop is reused, unchanged in shape, as the fallback target.
</objective>
