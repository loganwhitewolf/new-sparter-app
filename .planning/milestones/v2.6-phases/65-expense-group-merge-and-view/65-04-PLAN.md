<objective>
Wire the "Unisci" flow onto the expenses list: a gated bulk-action button, a merge dialog that
forces categorize-first for uncategorized selections, and expense-table row rendering that shows
a grouped row as a single link-to-group-detail row with an "Unita" badge instead of N member rows.

Purpose: this is the primary user-facing entry point for GRP-01/GRP-02, and the first place a
grouped row is actually seen and interacted with (GRP-03). It consumes `mergeExpenses` (Plan
65-02) and the composed `ExpenseRow.groupId/groupTitle` fields (Plan 65-03).
Output: modified bulk-action-bar, a new merge dialog, and a reworked expense table row renderer.
</objective>

    - Clicking the primary button in `'title'` step: if ANY entry in `selectedExpenses` has

    - In `'categorize'` step, selecting a subcategory via `SubcategoryPicker.onChange` calls the
      EXISTING `bulkCategorize` action scoped to ONLY the uncategorized ids (`selectedExpenses

      success the step becomes `'confirm'` (the dialog now treats every selected expense as
      sharing that category — the merge call in the next step includes the FULL original id set);
      on error, a toast shows the error and the step stays `'categorize'`.

      `selectedExpenses.map(e => e.id)` set and the entered `groupTitle`; on success, shows a
      success toast, calls `onSuccess()`, and closes the dialog (`onOpenChange(false)`); on error,
      shows `toast.error(result.error)` and stays open on the `'confirm'` step (never silently
      resets the user's title input).
    - The dialog never calls `mergeExpenses` before either confirming every selected expense
      already shares one non-null subCategoryId or completing the categorize step above — it can
      never reach the confirm step with mixed/uncategorized state (GRP-02).
