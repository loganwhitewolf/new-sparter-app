# Milestone v1.15: Collapsible Sidebar

**Status:** ✅ SHIPPED 2026-06-07
**Phases:** 41 (1 phase)
**Total Plans:** 3

## Overview

Replaced the two-zone topbar+sidebar layout with a single-zone collapsible icon-rail sidebar that owns all app chrome. The sidebar collapses to a 16-unit icon rail, persists its state in localStorage, and contains all navigation and user controls previously split between the topbar and sidebar. The topbar is completely removed on all breakpoints. Mobile users gained a 5th "Impostazioni" BottomNav entry. The ThemeToggle moved to a new Aspetto section in the SettingsHub.

## Milestone Summary

**Key Decisions:**

- `SidebarContext` type is exactly `{ collapsed: boolean; setCollapsed: (v: boolean) => void }` (D-13) — no extra fields
- localStorage key is `'sparter-sidebar-collapsed'` (D-05); SSR default is `false` (expanded) via `useState(false)` to prevent hydration mismatch (D-14)
- Tooltip rendered only when `collapsed && mounted` (queueMicrotask pattern) to avoid SSR tooltip flicker (Pitfall 6 from research)
- AppShell is a client component reading SidebarContext; RSC layout stays async server component — SidebarProvider bridges the RSC/client boundary
- `Impostazioni` removed from sidebar nav; settings accessible via user dropdown (desktop) and BottomNav 5th entry (mobile) + /settings route
- ThemeToggle moved from topbar to SettingsHub Aspetto section; component reused unchanged (D-12)
- `aria-label="Menu utente"` preserved on avatar trigger for E2E test compatibility
- ADR 0011 (docs/adr/0011-collapsible-sidebar-no-topbar.md): topbar permanently removed, no toggle to restore two-zone layout

**Issues Resolved:**

- React 19 runtime errors on sidebar collapse (hydration mismatch in tooltip rendering — mounted guard added)
- 2 pre-existing test failures in onboarding tests (R-OB-07 revalidation, R-OB-09 light theme on step 4) fixed as part of milestone cleanup

**Issues Deferred:**

- D-03 visual rendering (icon-only vs icon+label) — manual verification only; no unit test (E2E covers toggle flow)
- D-07 user name/email in user dropdown — manual verification only (needs authenticated session)
- D-09 active route CSS highlight — manual verification only
