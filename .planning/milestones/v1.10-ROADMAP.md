## Overview

End-to-end pattern suggestion pipeline for Sparter's import flow. Users can now discover recurring uncategorized bank descriptions, see ranked suggestions during import analysis, promote useful ones to categorization patterns before confirming the import, and re-run suggestion analysis after an import using persisted transactions — without touching the raw R2 file.

## Milestone Summary

**Key Decisions:**

- `detectPatternSuggestions` wrapped in isolated `try/catch` — detection failure must never block import (D-03)
- Dismissed suggestions are ephemeral — no dismissal persistence table (ADR + SCOP-01)
- Post-import DAL uses `innerJoin` on `importFile` — enforces ownership at query level; non-null `fileId` for all imported rows
- Suggestions capped at 5 sorted by `matchCount` desc — surface highest-signal patterns only
- `promoteSuggestionAction` confidence hardcoded at 0.85 — no UI knob; consistent with existing pattern confidence semantics
- Post-import page does not auto-reclassify: creating the pattern is the terminal action (SCOP-03)
- Partial-match-only rule (260524-pnk): suggestions emitted only when detector prefix ≠ full description
- `normalizeDescription` utility extracted for consistent uppercase+trim across analysis and suggestions pages
- `createPattern` handles unique-constraint violation by reactivating soft-deleted user patterns instead of throwing

**Issues Resolved:**

- Pattern not applied to uncategorized transactions after promotion from suggestions (260525-ga2)
- Pattern suggestions showing for exact-match descriptions — not useful signal (260524-pnk)

**Issues Deferred:**

- REVAL-01: Apply newly created pattern to existing transactions from same import — future milestone
- GLOBAL-01: Pattern suggestions across all uncategorized transaction history — future milestone
- DISM-01: Persistent dismissal — explicitly out of scope for v1.10

**Technical Debt:**

- ANL-01/02/03/04 checkbox and traceability table in REQUIREMENTS.md not updated until milestone close (execution tracking lag)
