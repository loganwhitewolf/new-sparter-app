<objective>
Create the client-side foundation for the collapsible sidebar: a React Context provider that
owns the collapsed boolean and persists it to localStorage (D-05, D-13, D-14), plus a shadcn-style
Tooltip wrapper (research-recommended) for icon-only nav items in collapsed mode.

Purpose: `app/(app)/layout.tsx` is an async RSC and cannot host React state. The provider is the
client island that all chrome components (the `<aside>` width and the Sidebar contents) read from.
This plan ships zero visual changes — it only adds reusable building blocks consumed by Plan 02.

Output: `components/layout/sidebar-provider.tsx`, `components/ui/tooltip.tsx`.
</objective>

  <behavior>
    - SidebarProvider renders children unchanged and supplies context value { collapsed, setCollapsed }
    - On first render (SSR + first client paint) collapsed === false (no localStorage read in initializer)
    - After mount, useEffect reads localStorage 'sparter-sidebar-collapsed'; if value is the string "true" collapsed becomes true, if "false" stays false, if null stays false (default expanded)
    - Calling setCollapsed(true) writes the string "true" to localStorage; setCollapsed(false) writes "false"
    - useSidebarCollapsed throws "useSidebarCollapsed must be used within SidebarProvider" when called outside the provider
  </behavior>
  <action>
    Create a "use client" module at components/layout/sidebar-provider.tsx exporting `SidebarProvider`
    (component) and `useSidebarCollapsed` (hook).

    Define the type `SidebarContextValue = { collapsed: boolean; setCollapsed: (v: boolean) => void }`
    matching D-13 exactly. Create a module-level `SidebarContext = createContext<SidebarContextValue | null>(null)`
    and a module-level constant `const STORAGE_KEY = 'sparter-sidebar-collapsed'` (D-05 — the exact key string).

    Inside SidebarProvider use `const [collapsed, setCollapsed] = useState(false)` (D-14: SSR default
    expanded, never read localStorage in the initializer — see Pitfall 1) and a `const [mounted, setMounted] = useState(false)`.
    In a `useEffect(() => { ... }, [])`: read `localStorage.getItem(STORAGE_KEY)`; if the stored value is
    not null, call the raw state setter with `stored === 'true'`; then call setMounted(true).

    Expose a wrapper setter `handleSetCollapsed = (v: boolean) => { setCollapsed(v); localStorage.setItem(STORAGE_KEY, String(v)) }`.
    Writing to localStorage after mount is safe because handleSetCollapsed only fires from user interaction
    (post-mount). Provide context value `{ collapsed, setCollapsed: handleSetCollapsed }`.

    useSidebarCollapsed calls `useContext(SidebarContext)`; if the result is null, throw
    `new Error('useSidebarCollapsed must be used within SidebarProvider')`; otherwise return it.

    Use developer-facing English for identifiers and comments per CLAUDE.md Language Convention.
    Do not use Decimal — no monetary values here.
  </action>
