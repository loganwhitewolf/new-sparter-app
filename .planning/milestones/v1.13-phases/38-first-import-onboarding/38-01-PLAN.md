<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authenticated browser → Next.js RSC layout | The layout receives an authenticated userId from `verifySession()` and queries `transaction` count restricted to that userId. No untrusted input crosses this boundary other than the `x-pathname` request header set by `proxy.ts`. |
| proxy.ts → request headers | `proxy.ts` writes `x-pathname` from `request.nextUrl.pathname` (Next.js parsed and normalized) into the forwarded request headers. A client cannot inject a forged `x-pathname` because Next.js strips inbound headers with this name OR `proxy.ts` overwrites it unconditionally. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-01 | Spoofing | `x-pathname` header read in `app/(app)/layout.tsx` | mitigate | `proxy.ts` overwrites `x-pathname` on every request via `requestHeaders.set('x-pathname', request.nextUrl.pathname)` — a client-supplied value is replaced before reaching the layout. |
| T-38-02 | Information Disclosure | `getTransactionCount(userId)` | mitigate | Query filters `WHERE userId = $1` using a userId obtained from `verifySession()` server-side; no userId from request body or query string. |
| T-38-03 | Information Disclosure | `getTopUncategorizedExpenses(userId, limit)` | mitigate | Parameterized SQL via Drizzle tagged template; user_id bound from session; sub_category_id and total_amount filters do not depend on user input. Wave 3 action layer must NOT accept userId from form data. |
| T-38-04 | Tampering | `getFileCoveredMonths(fileId, userId)` | mitigate | Inner join on `file` table with `eq(file.userId, userId)` enforces ownership — a guessed fileId from another user returns null. |
| T-38-05 | Denial of Service | `getTopUncategorizedExpenses` LIMIT clause | mitigate | Hard-cap `limit` parameter on the DAL side via `Math.min(limit, 100)` defensive cap; UI passes 15. |
| T-38-SC | Tampering | npm/pip installs | accept | No new packages introduced in this plan. Legitimacy gate not triggered. |
</threat_model>
