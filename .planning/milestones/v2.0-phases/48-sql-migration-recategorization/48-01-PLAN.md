    2. ADD COLUMN "nature_id" integer on "sub_category" and on "user_subcategory_override", each with the FK to nature(id) onDelete set null, plus the nature_id indexes (sub_category_natureId_idx, user_subcategory_override_natureId_idx).
    3. DROP the deprecated COLUMNS before dropping the enum types that back them: drop "sub_category"."nature" and "user_subcategory_override"."nature" (both flow_nature-typed) and "category"."type" (category_type-typed) and "categorization_pattern"."amount_sign" (amount_sign-typed). Column drops MUST precede the matching DROP TYPE.
    4. DROP INDEX "category_type_idx" before/with the category.type column drop.
    5. Reconcile the categorization_pattern unique constraint: the 0017 constraint is (pattern, sub_category_id, amount_sign); target is (pattern, sub_category_id). DROP CONSTRAINT "categorization_pattern_unique", drop the amount_sign column, then ADD the new 2-column "categorization_pattern_unique". Because dropping amount_sign can collapse two rows that previously differed only by sign into a (pattern, sub_category_id) duplicate, INSERT a collision-safe pre-dedup statement BEFORE the ADD CONSTRAINT: delete duplicate categorization_pattern rows keeping the lowest id per (pattern, sub_category_id) (e.g. DELETE ... WHERE id NOT IN (SELECT MIN(id) ... GROUP BY pattern, sub_category_id)). This dedup is the MIG-03 collision guard; document it inline with an English comment referencing D-07/MIG-03. This is the ONE data statement permitted in the migration — it exists solely so the constraint add cannot fail; all taxonomy/nature data transforms remain in seed-extras (D-05).
    6. DROP TYPE "flow_nature", "category_type", "amount_sign" AFTER their columns are gone.

    Preserve Drizzle's `--> statement-breakpoint` separators between every statement. Do not collapse statements. Keep the file ASCII/English in comments (Language Convention).

    Do NOT write a down-migration: column/enum drops are destructive and irreversible by SQL; reversibility is the pg_dump restore path (D-13), handled in the Plan 03 runbook.
