| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-02-01 | Tampering | `fetchMovers` direction parameter | mitigate | Validate `direction` against the closed set `['in','out','allocation']` before the DAL call (same bounds-check pattern as year/monthIndex on lines 24-33); reject with generic error otherwise |
| T-49-02-02 | Information Disclosure | direction-grouped aggregation queries | mitigate | `verifySession()` enforced at every cached DAL function entry; all WHERE clauses scope by `userId`; no cross-user leakage via the new direction join |
| T-49-02-03 | Tampering | direction join / WHERE clauses | mitigate | Direction filters use parameterized Drizzle (`eq(direction.code, ...)`, `ne(...)`) — no string-built SQL; the `direction.code` value compared is either a server-validated enum or a constant literal |
