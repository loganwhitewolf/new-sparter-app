<objective>
Restore direction as the category "type" across the categorization data layer (CAT-01, CAT-02, DASH-02). The critical deliverable is `getCategoriesForUser` re-deriving `CategoryWithSubCategories.type` from the `nature → direction` FK chain — currently a `null` stub that silently breaks cascade-options (it skips every category) and the SubcategoryPicker. Then re-point the Transactions and Expenses table filters from `nature.code`-as-type to a real `direction.code` filter, fix the `transactionListSelect.categoryType` placeholder (currently `category.id` — a wrong column), and clean up the smaller DAL stubs (`getMostUsedSubcategories` direction restriction, `getCategoryTypeForSubCategory` obsolete helper).

Purpose: this plan is the foundation the categorization UI (Plan 05) builds on. Per the research sequencing constraint, `getCategoriesForUser` must restore `type` BEFORE `cascade-options.ts` and the picker can work — so this DAL plan ships in Wave 1, the UI in Wave 2.
Output: rewritten `lib/dal/categories.ts`, `lib/dal/transactions.ts`, `lib/dal/expenses.ts`, `lib/dal/subcategory-usage.ts`, `lib/dal/patterns.ts`.
</objective>
