<objective>
Ship the data substrate for the income split: apply the enum migration to the live database, then add and run the additive seed-extras STEP that re-buckets the income-related subcategory slugs into `income_extraordinary` (D-03/D-04).

Purpose: The enum value must exist in the live DB before any query (`getOverviewChart`) consumes it, and the re-bucketing must run as a separate process AFTER `yarn db:migrate` so the `ADD VALUE` and its consuming UPDATE never share a transaction (Pitfall 1 / migration gotcha). The exact slug membership is a PO domain call confirmed in execution (D-04) — the data contract is fixed, the seed contents are tunable.

Output: A re-bucketing STEP appended to `scripts/seed-extras.ts`, the migration applied, and the seed run against the dev database.
</objective>
