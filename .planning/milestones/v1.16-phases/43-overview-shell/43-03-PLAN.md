<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → server component | `?year=` query param is the only attacker-controlled input; read server-side from `searchParams` |
| server component → DAL | `getOverview(year)` / `getOverviewChart(year)` receive the resolved numeric year |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-05 | Tampering / Injection | `?year=` → DAL queries | mitigate | `resolveYear` constrains the value to a member of `getYearsWithData()` (or the D-04 default); an arbitrary/hand-edited `?year=` never reaches the DAL verbatim. The Phase 42 DAL is parameterized (drizzle `sql` bindings), so even the numeric year is bound, not interpolated. |
| T-43-06 | Injection (reflected XSS) | year value rendered in copy | mitigate | The rendered year is the resolved number (or the `OverviewEmptyState` message); the raw `?year=` string is never echoed into the DOM. React escapes by default. |
| T-43-07 | Information disclosure | cross-user data via year scope | accept | `getYearsWithData`/`getOverview`/`getOverviewChart` all call `verifySession()` and scope to `userId` (Phase 42, T-42-05); the page adds no unscoped query. |
| T-43-SC | Tampering | npm installs | accept | No new package installs in this plan. |
</threat_model>
