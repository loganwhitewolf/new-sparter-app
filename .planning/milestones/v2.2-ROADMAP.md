## Milestone Summary

**Key Decisions:**

- ADR 0013: parsing contract (`delimiter`, `*Column`, …) moves from `platform` to `import_format_version`; behavior-preserving, regression-gated on 7 real CSV fixtures
- ADR 0014: per-bank PDF template (not generic); recognized by markers; normalized to `ParsedImportFile` with synthetic headers; sign via X coordinates (`unpdf`); balance chain as explicit guard
- Two-step migration strategy: ADD nullable columns → data copy via seed-extras → DROP old columns (migrations 0021 + 0022)
- `unpdf` chosen for positional X-coordinate extraction (serverless-ready); `pdf-parse` rejected (flat text, no coordinates)
- `UNRECOGNIZED_PDF_FORMAT` — single constant as both machine-readable code and Italian fallback message
- `PDF_IMPORT_PLATFORM_SLUGS` co-located with `.pdf` dispatch as single source of truth for PDF-capable banks
- No `fileType` column on `import_format_version` — allowlist approach avoids scope creep

**Issues Resolved:**

- 13 TypeScript compilation errors in regex-discovery.ts and seed-extras.ts (column references to dropped `platform` contract columns)
- Fused tipo+description PDF tokens corrupting row extraction (auto-fixed in 57-03 during E2E verification)
- UAT Test 5: unrecognized PDF produced internal error message leaking parser internals → fixed with enriched Italian UX (57-05)

**Issues Deferred:**

- Tier-1 categorization patterns for Trade Republic descriptions — follow-up via `regex-discovery` + `seed-patterns`
- PDF templates for additional banks (emerge from 2–3 concrete cases)
- Retroactive re-hash of already-imported history when `descriptionStripPattern` changes

**Technical Debt Incurred:**

- Migration 0022 requires backfill before merge of PR #24 (`yarn db:migrate → seed → seed-extras → seed-patterns`)
