# Milestone v2.1: Regex Discovery & Transaction Unification

**Status:** ✅ SHIPPED 2026-06-22
**Phases:** 51–55
**Total Plans:** 15

## Overview

Re-architected regex discovery as a separate step downstream of auto-categorization (previously it ran inside import, before categorization), removing duplicate and already-covered proposals, with a reusable trigger and a cleaned-up import summary. The discovery service is now standalone, operates exclusively on the uncategorized Set B, and is reachable from two entry points: automatic post-import and on-demand from the Files table.

## Phases

### Phase 52: regex-validity-and-dedup

**Goal**: The discovery service proposes a regex only for genuine prefix+variable families, surfaces identical-after-normalization groups as single categorizations instead, and never re-proposes something already covered.
**Depends on**: Phase 51
**Requirements**: RDISC-01, RDISC-02, RDISC-03, RDISC-04
**Plans**: 3 plans (complete 2026-06-16)

Plans:
- [x] 52-01: Pure util: descriptionHashes passthrough + candidateCoveredByExistingPattern (Check 1 helper, RDISC-03)
- [x] 52-02: DAL: getManuallyCategorizedHashes manual-history query (Check 2 data source, RDISC-04)
- [x] 52-03: Service: residual split + Check 1/Check 2 wiring + two-list DiscoveryResult (RDISC-01/02/03/04)

**Key deliverable**: Two-list `DiscoveryResult` — `candidates` (regex families with non-empty residual) and `singleCategorizationSuggestions` (identical-after-normalization groups).
