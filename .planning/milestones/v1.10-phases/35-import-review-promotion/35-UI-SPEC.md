## Typography

Inherits project-wide tokens. No new sizes introduced.

| Role | Size | Weight | Line Height | Class |
|------|------|--------|-------------|-------|
| Section heading | 16px (text-base) | 600 (font-semibold) | 1.4 | `text-base font-semibold` |
| Body / label | 14px (text-sm) | 400 (font-normal) | 1.5 | `text-sm` |

| Pattern string | 14px (text-sm) | 400 (font-normal) | 1.4 | `font-mono text-sm` |

Two weights only: `font-normal` (400) for body text, labels, captions, and mono pattern strings; `font-semibold` (600) for section headings.

Pattern strings use `font-mono` (Geist Mono) to signal machine-readable regex content. This aligns with how `CreatePatternDialog` already uses `font-mono` for the pattern field hint.

## Color

Inherits project CSS variables from `globals.css`. Colors below reference those tokens.

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `--background` | Page background, section container |
| Secondary (30%) | `--card` / `--secondary` | Suggestion cards (`bg-card`), form field backgrounds |
| Accent (10%) | `--primary` (green oklch 0.515 0.154 153.85) | "Crea pattern" submit button only |
| Destructive | `--destructive` (red oklch 0.637 0.237 25.33) | Inline error Alert only |
| Muted foreground | `--muted-foreground` | Caption text, sample description list, disabled state |
| Border | `--border` | Card borders, input outlines |

Accent (`--primary`) is reserved exclusively for: the "Crea pattern" submit button on each suggestion card.

**Success state:** Use `<Badge variant="outline">` with `className="border-green-600 text-green-700 dark:text-green-400"` for the "Pattern creato" badge. This avoids introducing a new CSS variable; the green family matches `--primary` (which is also green) but uses Tailwind's static green scale at `600/700` to ensure legibility without coupling to the theme ring color.

**Promoted card state:** When a suggestion is promoted, the card's inline form becomes `opacity-50 pointer-events-none`. The "Pattern creato" badge appears in the card header.

## States & Interactions

### SuggestionCard state machine

| State | Visual | Trigger |
|-------|--------|---------|
| Default | Form enabled, no badge | Initial render |
| Expanded samples | Sample list visible below toggle | User clicks "Mostra N esempi" |
| Collapsed samples | Sample list hidden | User clicks "Nascondi esempi" |
| Submitting | Button shows `<Loader2 animate-spin>`, disabled | `isPending === true` |
| Promoted | Form `opacity-50 pointer-events-none`, "Pattern creato" badge shown | `state.error === null` after submit |
| Error | `<Alert variant="destructive">` shown below selectors | `state.error !== null` |

### Sample toggle

- Uses local `useState<boolean>(false)` named `showSamples`.
- Button text: `"Mostra ${n} esempi"` when collapsed, `"Nascondi esempi"` when expanded.
- Sample descriptions render as `<ul class="flex flex-col gap-1">` with `<li class="text-xs text-muted-foreground truncate">`.
- Maximum 3 descriptions shown (cap enforced upstream in `sampleDescriptions` shape).

### Promotion feedback (useActionState pattern)

Follow the `useActionState` + `submittedRef` pattern from `CreatePatternDialog`:

```ts
const [state, formAction, isPending] = useActionState(promoteSuggestionAction, { error: null })
const submittedRef = useRef(false)

useEffect(() => {
  if (submittedRef.current && state.error === null) {
    setPromoted(true)
    submittedRef.current = false
  }
}, [state])
```

`setPromoted(true)` drives the promoted visual state (card form disabled, badge shown). No toast — the inline badge is the success feedback per D-02.

### Import confirm button

Remains visible and enabled regardless of suggestion state (REV-04). No changes to the existing confirm flow.

## Copywriting Contract

All UI copy is in Italian (product language convention).

| Element | Copy | Notes |
|---------|------|-------|
| Section title | `Suggerimenti pattern (N)` | N = `patternSuggestions.length`. Not rendered when N = 0. |
| Match count badge | `N match` | e.g. `3 match`. Badge variant="secondary". |
| Sample toggle — collapsed | `Mostra N esempi` | N = `sampleDescriptions.length`, max 3. |
| Sample toggle — expanded | `Nascondi esempi` | — |
| Category label | `Categoria` | Above category Select. `font-normal`. |
| Subcategory label | `Sottocategoria` | Above subcategory Select. `font-normal`. Matches `CreatePatternDialog`. |
| Category placeholder | `Categoria` | Inside SelectValue. |
| Subcategory placeholder | `Sottocategoria` | Inside SelectValue. Disabled until category selected. |
| Submit button — idle | `Crea pattern` | Button size="sm". |
| Submit button — pending | `Crea pattern` (+ spinner) | `<Loader2 class="mr-2 h-4 w-4 animate-spin" aria-hidden>` prepended. |
| Success badge | `Pattern creato` | Badge variant="outline" with green border/text. Appears in card header. |
| Validation error — no subcategory | `Seleziona una sottocategoria per creare il pattern.` | Returned from `promoteSuggestionAction` validation. Shown in Alert. |
| Validation error — invalid pattern | `Pattern regex non valido.` | From DAL call. Consistent with `createPatternAction` error. |
| Generic action error | `Si è verificato un errore. Riprova tra qualche secondo.` | From `promoteSuggestionAction` catch block. Consistent with `createPatternAction`. |
| Confirm button (no warnings) | `Conferma importazione` | Unchanged from existing `ImportPreview`. |
| Confirm button (with warnings) | `Conferma importazione (con avvisi)` | Unchanged from existing `ImportPreview`. |

No empty state copy needed: when `patternSuggestions` is empty, the entire `SuggestionSection` is omitted from the DOM (per D-04 / specifics note in CONTEXT.md).

No destructive actions in this phase. No confirmation dialogs required.

## Accessibility

- `<section aria-label="Suggerimenti pattern">` wraps the section for landmark navigation.
- Pattern `<code>` element has no extra aria; its role is decorative context.
- The sample toggle `<button>` uses `aria-expanded={showSamples}` and `aria-controls` pointing to the sample list id.
- Category and subcategory `<Select>` components must have `<label>` elements connected via `htmlFor` / `id` per the existing pattern in `CreatePatternDialog` (use `id` on `SelectTrigger`).
- Disabled subcategory Select when no category selected: `disabled` prop already handled by shadcn Select.
- Error Alert: `role="alert"` is already included in the existing `Alert` component pattern (see `ImportPreview` confirm error block).
- "Pattern creato" badge: no additional aria; the promoted state makes the form `pointer-events-none` which is sufficient for keyboard users since the button is also `disabled`.
