<objective>
Extend upload validation so Trade Republic PDF statements can be uploaded via the existing presigned PUT path. Add application/pdf (and a defensive application/octet-stream fallback) to the MIME allowlist, .pdf to the extension allowlist, and surface .pdf in the uploader UI. The 5 MB cap already exists and must continue to reject oversized files before the presigned URL is created.

Purpose: PDF-01 — the user can upload a .pdf via presigned PUT with a 5 MB cap that fails with an explicit error. The page ceiling is a separate concern enforced at parse time (the initiate route has only name/size/type, not bytes), so this plan delivers the upload-gate half of PDF-01.
Output: PDF uploads accepted through the same presigned PUT flow as CSV/XLSX; oversized files still rejected pre-upload.
</objective>
