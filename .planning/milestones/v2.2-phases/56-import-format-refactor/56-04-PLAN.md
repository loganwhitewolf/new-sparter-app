<objective>
Re-point every runtime consumer to read the parsing contract from `import_format_version` instead of `platform`, completing the behavior-preserving refactor (D: ADR 0013 consumers list; IFMT-05 consumers, IFMT-04 versioning expressible). The detection DAL join, the candidate config type, `scoreCandidate`, and the private-format wizard all move off `format.platform.*` and onto the format-version-sourced contract. The import service (`lib/services/import.ts`) consumes the contract only through the candidate's config object, so updating the candidate shape here re-points it without editing import.ts.

The phase invariant proof lands in this plan: after the change, the Plan 01 pinned regression test and the detector fixture test must both be GREEN (IFMT-02). Test files are updated only where they hard-coded the contract on the platform sub-object (e.g. tests/import-detector.test.ts builds `formats` with the contract under `platform`); update those fixtures to the new candidate shape WITHOUT changing the expected detection outcomes or hashes.

Purpose: Finish the move so versioning works end to end with zero behavioral change.
Output: detector/DAL/type/wizard reading the contract from the format version; both regression tests GREEN.
</objective>
