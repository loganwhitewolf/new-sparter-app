<objective>
Behavior-preserving glue to keep the build green and the wizard write-path consistent after the platform column rename/drop (D-06 / PLAT-01). Adapt `createPrivateRows()` in `lib/services/import-format-wizard.ts` to the new platform schema: stop writing the dropped `visibility` column, rename the platform insert key from `ownerUserId` to `proposedByUserId`, and set the new platform's `reviewStatus` to `'pending'` (the lifecycle value) instead of `'draft'`. Update the wizard test assertions to match.

Purpose: Dropping `platform.visibility` and renaming `ownerUserId` breaks the wizard's platform insert (a write to the gone `visibility` column is a runtime error; the `ownerUserId` insert key is now a TypeScript error). This is minimal, behavior-preserving adaptation — NOT the Phase 59 attach-format UX. Aligning `reviewStatus` to `'pending'` ensures a freshly created private platform is actually visible to its proposer under the relaxed `accessibleWhere` (Plan 02), where `'draft'` would not match the lifecycle guard.

Output: An adapted `createPrivateRows` and matching wizard-actions test assertions.
</objective>
