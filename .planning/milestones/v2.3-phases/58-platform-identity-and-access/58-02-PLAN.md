<objective>
Decouple a private Import Format from a private Platform and wire the `reviewStatus` visibility lifecycle into the import-format access query. Relax `lib/dal/import-formats.ts` `accessibleWhere()` so a user-owned `importFormatVersion` is visible to its owner on ANY platform the user may see (global/approved included), and make platform visibility key on `reviewStatus` (`pending` → proposer-only, `approved` → all) — with NO regression for the existing global formats (D-03, D-04, D-05 / PLAT-02, PLAT-03, SC2, SC3, SC4).

Purpose: This is the security boundary of the phase (ASVS V4 — broken access control / IDOR). The relaxation must not over-expose another user's private format nor hide global-approved formats. The DAL filters in TWO layers (SQL WHERE + in-memory fail-closed re-check) that must change in lockstep, or the in-memory guard silently returns `[]` while unit tests stay green.

Output: A relaxed, lockstep-consistent `lib/dal/import-formats.ts` and an extended visibility-matrix unit test covering owner / non-owner / cross-user / pending-platform cases plus the existing global-approved path.
</objective>
