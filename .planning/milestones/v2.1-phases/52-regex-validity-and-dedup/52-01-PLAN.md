<objective>
Extend the pure clustering util `lib/utils/pattern-suggestions.ts` with two additive capabilities that the Phase 52 service (Plan 03) consumes:

1. **descriptionHashes passthrough** — carry `descriptionHash` per detector row and collect `descriptionHashes: string[]` per emitted suggestion, so Check 2 (Plan 03) can do a clean set-membership test against the user's manual-history hashes. Mirrors the additive-passthrough pattern Phase 51 established for `rawTitle` / `strippedByNormalization`.

2. **candidateCoveredByExistingPattern helper (Check 1, RDISC-03)** — a pure helper that tests whether a surviving candidate's GENERATED regex would already be covered by an existing active pattern, reusing the exact dual full + numeric-stripped, case-insensitive matcher used by `isCoveredByPatterns` / `applyTier1Regex` (matcher fidelity). This is NOT the Phase 51 input-row coverage filter (that filter stays) — it is the explicit generated-candidate gate RDISC-03 mandates.

Purpose: Provide the two pure building blocks the service needs, keeping all DB-free / script-safe logic in the util where it belongs (Architectural Responsibility Map: SC-1/SC-2 + Check 1 are Pure-util tier).
Output: Extended `PatternDetectorRowWithMeta` / `PatternSuggestionWithMeta` interfaces (one new field each), one new exported helper, and unit tests proving the empty-residual classification, the hash passthrough, and the Check 1 helper.
</objective>
