## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-53-04 | Elevation of privilege | Cross-user fileId in promote form | mitigate | `getPlatformIdForUserFile` WHERE includes `eq(file.userId, userId)`; null → error, no pattern write (ASVS V4). |
| T-53-05 | Tampering | Forged client platformId | mitigate | Derive platform from file server-side; do not read `platformId` from FormData as authority (53-RESEARCH Pattern 4). |
| T-53-06 | Tampering | Mass categorization via malicious regex | mitigate | Existing `CreatePatternSchema` / `normalizePatternInput`; platform scope bounds blast radius. |
| T-53-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs. No checkpoint needed. |
