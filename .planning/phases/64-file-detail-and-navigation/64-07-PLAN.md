<objective>
Second-round gap closure for Phase 64. Re-verification (2026-07-06T17:00:00Z) found one
blocking gap via code review — CR-01: the inline-edit pencil icon on all three detail pages
(`/transactions/[id]`, `/expenses/[id]`, `/import/[fileId]`) is permanently invisible because its
`opacity-0 group-hover:opacity-100` utility has no `.group` ancestor on any detail page, even
though the identical component works correctly inside table rows (where `TableRow` already
carries `.group`). This breaks DET-08's "displayName editable inline" promise: the feature only
works if a user clicks the exact invisible hitbox by chance.

The user also approved bundling WR-02 (a related, non-blocking review finding) into this same
round because it shares `detail-page-shell.tsx` and threatens the currently-pending human
verification for DET-09 (smart-back filter/scroll preservation): `handleBackClick`'s
`isExternalReferrer` check reads `document.referrer`, which is set once at the browser's initial
hard navigation and never updated by Next.js App Router client-side transitions — so any tab that
has ever loaded from an external referrer silently disables smart-back (falls back to
`router.push`, losing the origin table's filters) for the rest of its lifetime, even when
`window.history.length > 1` and genuine in-app history exists.

Purpose: make DET-08's inline-edit affordance actually discoverable, and make DET-09's smart-back
heuristic reliable regardless of a tab's navigation origin — both fixes compose with the existing
Plan 64-06 Client-Cache-busting logic without duplicating or reverting it.

Output: `.group` ancestor added to exactly the three correct wrapper divs (no change to the
shared pencil components themselves); `handleBackClick` branches solely on a new
`hasInAppHistory` pure helper, with `document.referrer`/`isExternalReferrer` fully removed.
</objective>
