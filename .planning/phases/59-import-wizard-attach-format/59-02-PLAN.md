<objective>
Add the backend write path for attaching a private Import Format to an existing Platform (D-06), plus the server-action wrapper that feeds step 1's platform list to the RSC page (D-02, D-07 consumer).

Purpose: PLAT-04 requires reusing an existing Platform when one fits (no duplicate rows) and minting a new one only when none does. The single fork point is `createPrivateRows`; the Zod schema and action must carry the new optional `existingPlatformId` through to it.
Output: optional `existingPlatformId` in the Zod schema with a `platformName` guard, the attach/create fork in `createPrivateRows`, the action propagating `existingPlatformId`, and `listAttachablePlatformsAction`. Test coverage for attach, create regression, and Zod optionality.
</objective>
