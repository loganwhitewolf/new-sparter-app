## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser â†’ Server Action | User-submitted nature value crosses into server scope; must be validated against the FlowNature enum before reaching the DAL. |
| Server Action â†’ DAL | userId is bound by `verifySession()`, not from the form. subCategoryId is user-supplied but constrained via DB foreign-key (cascade on delete) â€” invalid IDs surface as upsert failures. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-05-01 | Tampering | nature value via form submission | mitigate | `NatureSchema = z.enum([...])` rejects any value not in the 6-literal allowlist; `null` only accepted via the explicit `.nullable()` on SetSubcategoryNatureSchema. ASVS L1 V5 satisfied. |
| T-37-05-02 | Tampering | setSubcategoryNatureAction targeting another user's override row | mitigate | Action calls `verifySession()` â†’ userId; DAL upsert keys on (userId, subCategoryId), so an attacker cannot mutate another user's row. ASVS L1 V4 satisfied. |
| T-37-05-03 | Tampering | createSubcategoryAction missing nature | mitigate | CreateSubcategorySchema makes nature REQUIRED; safeParse returns failure before any DB write. |
| T-37-05-04 | Information Disclosure | effectiveNature leak via getCategoriesForUser | mitigate | DAL already user-scoped (Plan 37-03); effectiveNature is per-user by construction. |
| T-37-05-05 | Denial of Service | toast spam / repeated toggle | accept | UI button is disabled during the transition; no server-side amplification. |
| T-37-05-SC | Tampering | npm installs | mitigate | No new package installs. |
</threat_model>
