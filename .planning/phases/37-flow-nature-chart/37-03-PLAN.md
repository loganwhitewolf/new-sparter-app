## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Session → DAL | `getMonthlyTrendByNature` and `getCategoriesForUser` both gate on `verifySession()`; userId is bound at session-resolution time, not from client input. |
| DAL → DB | Drizzle parameterizes all values; the only client-derived input is the `preset` enum (validated by `dashboardPresetToDateRange`). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-03-01 | Tampering | `getMonthlyTrendByNature` userId scope | mitigate | `verifySession()` returns userId; userSubcategoryOverride JOIN clause hardcodes `userId = $userId` so cross-user override leakage is impossible. ASVS L1 V4 satisfied. |
| T-37-03-02 | Information Disclosure | Nature override leak across users | mitigate | Same JOIN clause; LEFT JOIN with userId predicate prevents another user's override from contaminating the COALESCE result. |
| T-37-03-03 | Tampering | preset parameter | mitigate | `dashboardPresetToDateRange` validates preset against a known enum; invalid values throw before query execution. ASVS L1 V5 satisfied. |
| T-37-03-04 | Spoofing | Drizzle sql template injection | accept | `coalesce(${userSubcategoryOverride.nature}, ${subCategory.nature})` uses Drizzle column references (compile-time), not interpolated strings; no SQL injection surface. |
| T-37-03-SC | Tampering | npm installs | mitigate | No new package installs. |
</threat_model>
