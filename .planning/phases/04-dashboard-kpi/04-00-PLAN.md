Create `lib/utils/date.ts` with these exports:

```typescript
export const DASHBOARD_PRESETS = [
  'last-month',
  'last-3-months',
  'last-6-months',
  'this-year',
  'last-year',
] as const

export type DashboardPreset = (typeof DASHBOARD_PRESETS)[number]
export type ExpensePeriod = 'this-month' | 'last-3-months' | 'last-6-months' | 'this-year' | 'last-year'
export type DateRange = { from: Date; to: Date }

export function periodToDateRange(period: ExpensePeriod | string, now = new Date()): DateRange
export function dashboardPresetToDateRange(preset: DashboardPreset | string, now = new Date()): DateRange
export function monthKey(date: Date): string
export function monthLabel(month: string): string
export function monthsBetween(from: Date, to: Date): string[]
```

Rules:
- `periodToDateRange('this-month')` preserves the exact current behavior from `lib/dal/expenses.ts`.
- `dashboardPresetToDateRange('last-month')` returns the previous calendar month, not the current month.
- `dashboardPresetToDateRange('last-3-months')` returns from the first day two months before `now` through the end of the current month.
- `monthKey()` returns `YYYY-MM`.
- `monthLabel()` formats `YYYY-MM` as an Italian short month such as `Apr`.

Update `lib/dal/expenses.ts`:
- Remove the local `periodToDateRange` function body.
- Add `import { periodToDateRange } from '@/lib/utils/date'`.
- Keep `periodToDateRange` available to existing consumers by re-exporting it:

```typescript
export { periodToDateRange } from '@/lib/utils/date'
```

Create `lib/utils/dashboard.ts` with these exports:

```typescript
import Decimal from 'decimal.js'
import { toDecimal } from '@/lib/utils/decimal'

export function computeSavingsRate(totalIn: string | number, totalOut: string | number): number
export function computeDeltaPercent(current: string | number, previous: string | number): number | null
export function computeBreakdownPercentages<T extends { count: number }>(rows: T[]): Array<T & { percentage: number }>
```

Rules:
- `computeSavingsRate(0, anything)` returns `0`.
- Otherwise savings rate is `(totalIn - totalOut) / totalIn * 100`, rounded to one decimal.
- `computeDeltaPercent(current, 0)` returns `null`.
- Otherwise delta is `(current - previous) / abs(previous) * 100`, rounded to one decimal.
- `computeBreakdownPercentages([])` returns `[]`.
- Percentages are count-based for Phase 4 because no transaction amount column exists yet.
