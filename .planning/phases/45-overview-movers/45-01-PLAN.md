## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Server Action (`fetchMovers`) | `year` and `monthIndex` arrive from client state on bar click; untrusted integers cross here |
| Server Action → DAL | DAL re-scopes every query to the authenticated `userId` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-01 | Tampering | fetchMovers(year, monthIndex) inputs | mitigate | `Number.isInteger` bounds: year 2000–2100, monthIndex 0–11; out-of-range returns `{ movers: [], error }` without hitting the DAL |
| T-45-02 | Information Disclosure / Elevation | fetchMovers reaching another user's data | mitigate | `verifySession()` called first at the action trust boundary; DAL `getMonthOverMonthCategoryChanges` independently scopes by `userId` (defense in depth) |
| T-45-03 | Repudiation | read-only action with no side effects | accept | No writes, no DB mutation, no revalidation; nothing to log beyond existing auth |
| T-45-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan; nothing to verify |
</threat_model>
