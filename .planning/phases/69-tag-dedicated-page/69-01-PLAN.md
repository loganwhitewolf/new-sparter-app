Wire ONE path through every layer the phase touches, end-to-end, for the single "open a tag's page" capability (D1, D4, D5). No breakdown card yet — that is Plan 69-02.

1. lib/routes.ts — add a `tagDetail(id: number | string)` helper next to the existing `dashboardCategoryDetail`/`transactionDetailHref` helpers, returning `${APP_ROUTES.tags}/${encodeURIComponent(String(id))}`. This is the single source for every link into the page (D1).

2. app/(app)/tags/[id]/page.tsx — new authenticated RSC (Next.js 16 async `params: Promise<{ id: string }>`, mirror app/(app)/dashboard/categories/[id]/page.tsx exactly for the await/parse shape). Steps: `verifySession()` for `userId`; parse the `id` param with a local positive-int guard (regex `^\d+$`, then `Number.isSafeInteger` and `> 0`) — malformed → `notFound()`; `getTag(userId, id)` ownership check — `null` → `notFound()` (this is the IDOR boundary, D1); `getTagDetail(userId, id)` for the numbers. Render a page shell (`flex flex-col gap-6 p-6`, `max-w-4xl` container) with a header: tag name (`h1`), the date-range label via a `formatDateRange(tag)` helper returning "Nessun intervallo" when either bound is null (identical wording to the current panel, D4), an "Archiviato" badge when `tag.archived`, and — in a right-aligned action group — `EditTagDialog` plus `ArchiveTagDialog` rendered only when NOT archived (D5, mirror the current panel's conditional). Below the header render `<TagDetailReport detail={detail} />`. Keep a static `metadata = { title: 'Tag' }` (a dynamic per-tag title via generateMetadata is Claude's discretion).

3. components/tags/tag-detail-report.tsx — new presentational SERVER component (no "use client"; it takes props and does pure formatting, so it must not be a client component). Props: `{ detail: TagDetail }`. Lift the it-IT currency/date formatters, the sign-based tone helper (`>= 0 ? --total-in : --total-out`), the absolute/signed amount formatters and the count label from the current TagSettingsPanel TagDetailView (they are the locked D4 presentation rules). Render, top-to-bottom: (a) a 3-card grid (`sm:grid-cols-3`) of shadcn Card KPIs — "Entrate" = absolute inflow in `--total-in`, "Uscite" = absolute outflow in `--total-out`, "Valore finale" = signed `detail.net` colored by its sign; all amounts `tabular-nums`; (b) a muted "{n} transazioni incluse" line (singular "1 transazione inclusa") using `detail.count`; (c) the compact transaction list — when empty, the muted "Nessuna transazione inclusa per questo tag." copy; otherwise a scrollable (`max-h-[420px] overflow-y-auto`) bordered `divide-y` list, one row per `detail.transactions` item showing date (it-IT, `tabular-nums`, muted) · subcategory (truncate) · signed amount (tone by sign, `tabular-nums`), already date-descending from the DAL (TAG-10). Sign-color, tabular-nums and it-IT formatting per D4.

4. tests/tag-detail-report.test.tsx — static-render smoke test (renderToStaticMarkup + createElement, the repo's no-jsdom pattern used in tests/tag-settings-panel.test.tsx). With a fixture `TagDetail` (a couple of transactions, one in / one out, a non-zero net), assert the rendered HTML contains the three KPI labels, the "N transazioni incluse" count line matching the fixture length (TAG-08), each fixture subcategory name, and the it-IT-formatted net. Add a second case with `transactions: []` asserting the empty-state copy.
