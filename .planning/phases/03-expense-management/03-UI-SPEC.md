## Typography

All text uses Geist Sans unless noted. Numbers in tables use Geist Mono.

**Weights: exactly 2 — 400 (normal) and 600 (semibold).**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Table cell content, filter labels, form helper text |
| Label | 14px (text-sm) | 400 (normal) | 1.4 | Form field labels; column headers use `uppercase tracking-wide text-xs` to differentiate visually |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Page title "Spese"; modal dialog title |
| Display | 28px (text-3xl) | 600 (semibold) | 1.2 | Not used in Phase 3 — reserved for dashboard KPI |

Label differentiation: column headers are styled `text-xs uppercase tracking-wide text-muted-foreground font-normal` — visual distinction is achieved via size + case + color, not weight.

Mono override (apply `font-mono` class):
- Expense title in table cells (normalised bank descriptions, often uppercase) — Geist Mono 14px/400, `tracking-tight`, `uppercase` if source is all-caps, `truncate` after 48ch
- `createdAt` date column — Geist Mono 13px/400, `tabular-nums`
- Row count in floating action bar label — Geist Mono inline

| Role | CSS Token | Oklch Value | Usage |

| Destructive | `--destructive` | oklch(0.637 0.237 25.33) — red-500 | Delete action only |

Accent (`--primary` / emerald-600) is reserved for exactly these elements:
1. "Nuova spesa" Button (primary variant) in page header
2. "Categorizza (N)" Button in floating action bar (primary variant)
3. "Salva spesa" / "Conferma" submit Button inside modals (primary variant)
4. Checkbox checked state (inherits `--primary` via shadcn ring)
5. Focus ring on Select, Input, Textarea (via `--ring: var(--primary)`)

Status badge colors (from 03-CONTEXT D-06 — use Badge component with custom variant classes):

| Status | Value | Badge background | Badge text | When |
|--------|-------|-----------------|------------|------|
| Da categorizzare | status = 1 | `bg-amber-100` | `text-amber-700` | No subCategoryId assigned |
| Categorizzata | status = 2 or 3 | `bg-emerald-100` | `text-emerald-700` | subCategoryId assigned (auto or manual) |

Implementation: use `<Badge>` with `variant="outline"` overridden via `className` — do NOT introduce new Badge variants into the component file. Keep variant extension in the consuming component only.

## Modal: Create / Edit Expense

Trigger: "Nuova spesa" Button (page header) for create; "Modifica" in DropdownMenu for edit.

Dialog width: `max-w-md` (448px).

| Field | Component | Validation | Placeholder |
|-------|-----------|------------|-------------|
| Titolo * | `<Input>` | Required, min 2 chars, max 120 chars | "es. Netflix, Amazon Prime..." |
| Categoria > Subcategoria | `<Select>` (two-level: first category, then subcategory) | Optional | "Seleziona una categoria" |
| Note | `<textarea>` (styled) | Optional, max 500 chars | "Aggiungi una nota..." |

Hierarchy Select: single `<Select>` with `<SelectGroup>` per category, `<SelectItem>` per subcategory. Category name is a `<SelectLabel>` (non-selectable). Users pick a subcategory, not a parent category.

Modal footer: `[Annulla]` (ghost) — `[Salva spesa]` (primary) for create; `[Annulla]` (ghost) — `[Aggiorna spesa]` (primary) for edit.

## Dialog: Bulk Categorization

Trigger: "Categorizza (N)" in floating action bar.

Dialog width: `max-w-sm` (384px).

Content:
- Body: "Assegna una categoria a **N spese** selezionate."
- `<Select>` — same two-level category/subcategory picker as the form modal
- Footer: `[Annulla]` (ghost) — `[Conferma]` (primary, disabled until category selected)

On confirm: Server Action `bulkCategorize` sets `status = 3` and `subCategoryId` for all selected expense IDs. Dialog closes. Selection is cleared. Table revalidates.

## Interaction Contracts

### URL State (D-09)
Filters persist in URL search params. On filter change: `router.push('/spese?' + params.toString(), { scroll: false })`.

Default URL state (no params): all categories, all statuses, "Questo mese".

| Param | Values | Default |
|-------|--------|---------|
| `category` | category slug or omitted | omitted (all) |
| `status` | `uncategorized` \| `categorized` \| omitted | omitted (all) |
| `period` | `this-month` \| `last-3-months` \| `last-6-months` \| `this-year` \| `last-year` | `this-month` |

### Checkbox Behavior
- Header checkbox: `indeterminate` state when some-but-not-all rows checked. Clicking cycles: none → all → none.
- Floating action bar: appears with `transition-all duration-150` slide-up animation when `selectedCount >= 1`; disappears when `selectedCount === 0`.
- Selection is cleared on: successful bulk categorize, page navigation, filter change.

### Loading States
- Filter change: table rows show `opacity-50` skeleton shimmer while revalidating (use `isPending` from `useTransition`).
- Form submit: Salva spesa/Aggiorna spesa/Conferma button shows `<Loader2 className="animate-spin" />` icon + disabled state while Server Action is pending.
- Delete: row shows `opacity-50` while deletion is pending; removed from list on success.

### Responsive Behavior
- `md` breakpoint (768px): sidebar visible. Table shows all columns.
- `< md` (mobile): sidebar collapses to bottom nav. Table collapses: hide "Categoria" column, show only Titolo + Stato Badge + three-dots menu. Floating action bar remains visible above bottom nav.
