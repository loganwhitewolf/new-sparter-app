<objective>
Extend the pure clustering util `lib/utils/pattern-suggestions.ts` with an ADDITIVE WithMeta variant that emits the D-05 per-candidate normalization metadata required by PIPE-03. The existing `detectPatternSuggestions` function and `PatternSuggestion` type are left untouched so the current `analyzeFile` caller in `lib/services/import.ts` keeps compiling unchanged (per D-02 and RESEARCH "additive extension").

Purpose: PIPE-03 requires that discovery report what platform normalization already collapsed versus the residual-variable text the regex must still handle. That report is per-candidate metadata (D-05), produced here at the clustering layer where the stable prefix and residual variable part are computable. This is the script-safe foundation the Wave 2 service consumes.

Output: New exports `PatternDetectorRowWithMeta`, `PatternSuggestionWithMeta`, `detectPatternSuggestionsWithMeta` in `lib/utils/pattern-suggestions.ts`, plus a unit test file proving the metadata fields are populated correctly.
</objective>
