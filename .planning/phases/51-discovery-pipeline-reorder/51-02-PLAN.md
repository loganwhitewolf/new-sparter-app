<objective>
Create the DAL query `getUncategorizedExpensesForDiscovery(userId, platformId)` in a new file `lib/dal/regex-discovery.ts`. It fetches the persisted uncategorized set (Set B) for one user, scoped to a single platform via the standard expense→platform join chain, and returns each expense's `title`, `descriptionHash`, and the platform's `descriptionStripPattern`. This is the data source for PIPE-01 and D-01 (discovery runs post-categorization on the persisted Set B — `subCategoryId IS NULL` after `categorizePipeline` has written results): discovery reads only what auto-categorization could not classify, never Set A.

Purpose: PIPE-01 requires discovery input to be the still-uncategorized residual only, excluded from another user's or another platform's data. The `subCategoryId IS NULL` filter is the authoritative Set B signal (mirrors `applyNewPatternToExpenses`), and the `platform.id` filter enforces the D-03 platform-bounded read scope and the security requirement that discovery never crosses into another platform's or user's history.

Output: New file `lib/dal/regex-discovery.ts` exporting `getUncategorizedExpensesForDiscovery` and `UncategorizedExpenseForDiscovery`, plus a unit test that asserts the WHERE conditions and join chain via the codebase's mocked-db-query-chain pattern.
</objective>
