<objective>
Build the Trade Republic PDF parser as a pure service that converts PDF bytes into a ParsedImportFile. It recognizes TR by document markers, extracts only the TRANSAZIONI SUL CONTO section, attributes amount signs by X-coordinate position, validates the running-balance chain with Decimal.js, enforces the page ceiling, and emits synthetic headers matching the seeded import_format_version.

Purpose: PDF-02 (marker recognition + section isolation), PDF-03 (positional sign + balance chain), PDF-01 (page ceiling), and the parser side of PDF-05 (descriptionStripPattern aggregation is exercised here against the seeded pattern). This is the heaviest plan — a single focused module with its own test file.
Output: lib/services/trade-republic-pdf-parser.ts exporting parseTradeRepublicPdf(bytes, options) -> Promise<ParsedImportFile>, fully tested against the Wave 0 fixture.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| R2 object bytes → parser | Untrusted PDF bytes are parsed server-side by unpdf; malicious/oversized PDFs must not crash or OOM the function |
| extracted numbers → import | Positionally-extracted amounts/balances are untrusted until the balance chain certifies them |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-57-03-01 | Tampering | balance chain | high | mitigate | validateBalanceChain with Decimal.js certifies prev+amount==curr per row; any mismatch returns explicit error and zero rows (no partial/wrong import) — per D-03 |
| T-57-03-02 | Denial of Service | unpdf parse | high | mitigate | MAX_PDF_PAGES (50) checked via pdf.numPages before row extraction; 5 MB cap already enforced upstream in Wave 1 |
| T-57-03-03 | Tampering | malicious PDF JS | medium | accept | unpdf serverless PDF.js disables eval (isEvalSupported false) by default; no script execution from PDF content |
| T-57-03-04 | Information Disclosure | SSRF via embedded URL | low | accept | unpdf does not follow URLs; extraction is pure content parsing |
</threat_model>
