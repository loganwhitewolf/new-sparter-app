Add the four Phase 66 server actions — `categorizeExpenseGroup` (GRP-05), `addExpensesToGroupAction`
(GRP-06), `removeExpenseFromGroupAction` and `dissolveExpenseGroupAction` (GRP-07) — to
`lib/actions/expenses.ts`. Each is a thin `"use server"` orchestration wrapper: parse FormData

`revalidateCategorizationSurfaces()`.

(Plan 66-03's invariance test, Plan 66-04's table/dialog UI, Plan 66-05's group detail UI) calls
these four exports — no other new backend entry point is needed.
