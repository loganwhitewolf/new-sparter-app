<objective>
Thread an optional trailing `tagId` parameter through the five `lib/dal/dashboard.ts` exports that
power the Categories tab and its category drill-down sub-route (`getUncategorizedCount`,
`getOverviewAmountTotals`, `getCategoryRanking`, `getCategoryDeviations`, `getCategoryDetail`),
using the `tagScopedTransactions()` predicate from Plan 68-01. This is TAG-04's "every existing
widget narrows" requirement applied to the Categories tab and — per the phase's RESOLVED Open
Question #2 — its drill-down page too.

Purpose: `lib/dal/overview.ts`'s `getOverview` (Plan 68-03) calls `getOverviewAmountTotals` and
`getUncategorizedCount` directly, so those two signatures must land here first. The Categories
tab RSC pages (Plan 68-06) call `getCategoryRanking`/`getCategoryDeviations`/`getCategoryDetail`
directly.

Output: Five updated exports in `lib/dal/dashboard.ts`, each narrowing correctly by tag while
staying fully backward-compatible when no `tagId` is passed.
</objective>
