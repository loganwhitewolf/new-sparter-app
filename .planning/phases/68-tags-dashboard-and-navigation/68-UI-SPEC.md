# Phase 68 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Requirements covered:** TAG-04 (global dashboard tag filter), TAG-05 (Tag section — independent per-tag totals + archive), NAV-01 (dashboard row → filtered transactions click-through).

**Precedent this phase must not fork from:** Phase 67 shipped `/settings/tags` (CRUD), tag chips (`Badge variant="outline" text-[10px]`), the archived-badge idiom (`Badge variant="secondary" text-[10px]` "Archiviato" — never hidden/disabled), and `ArchiveTagDialog` (destructive-variant confirm, copy locked below). This phase is pure UI-layer wiring on top of that vocabulary — no new chip/badge/dialog visual language.

## Typography

Sizes and weights are the exact set already load-bearing across `OverviewHeader` (`text-lg font-semibold`), `DashboardCategoriesPage` (`text-xl font-semibold`), `CategoryRankingList` (`text-sm font-semibold` name, `text-xs text-muted-foreground` meta), and `OverviewMoversPanel` (`text-xs` rows). This phase reuses them as-is — no new sizes.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label / meta | 12px (`text-xs`) | 400 (regular) | 1.5 |
| Heading (section) | 18px (`text-lg`) | 600 (semibold) | 1.2 |
| Display (page title) | 20px (`text-xl`) | 600 (semibold) | 1.2 |

Amount/total figures always render `font-mono tabular-nums` at the weight of their role (semibold for the primary per-tag total, regular for meta counts) — existing convention, not a new size.

Note: `font-medium` (500) exists on pre-existing nav/tab active states (`DashboardTabNav`, `SortToggle`) — a pre-existing exception, not extended to any new component in this phase.

Existing 60/30/10 UI-chrome contract (unchanged, sourced from `app/globals.css`):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch(1 0 0), near-white) | Page background |
| Secondary (30%) | `--card` / `--muted` | Tag-section card surfaces, filter-row/empty-state backgrounds, `odd:bg-muted/30` movers-row striping |
| Accent (10%) | `--primary` (green, oklch(0.515 0.154 153.85)) | Reserved for: active `DashboardTabNav` tab underline (extends to the new "Tag" tab), the tag-filter `SelectTrigger`'s focus ring, hover/focus-visible state on newly-clickable movers rows and Tag-section card title links (`hover:underline`, `focus-visible:ring-2 focus-visible:ring-ring`) — same treatment as `CategoryRankingList`'s existing title link. Never used for large fills. |
| Destructive | `--destructive` (red, oklch(0.637 0.237 25.33)) | Reserved for: the Tag-section "Archivia" button + `ArchiveTagDialog` confirm (reused verbatim from Phase 67 — no new destructive surface introduced) |

Accent reserved for: active dashboard tab indicator, tag-filter select focus/active state, clickable-row/card hover-and-focus affordance. Never for static badges, never for the tag chips themselves (those stay `outline`/`secondary` per Phase 67).

**Separate pre-existing data-semantics palette (not part of the 60/30/10 chrome budget — reused, not introduced):** `--total-in` (green), `--total-out` (red), `--total-allocation` (`#a78bfa`), `--total-transfer` (gray), `--balance`. The Tag-section per-tag total reuses the exact sign-based tone rule already implemented in `OverviewMoversPanel`'s allocation column (`Number(total) >= 0 ? 'text-[var(--total-in)]' : 'text-[var(--total-out)]'`) — a tag can contain both inflow and outflow transactions, so its net total is colored by sign, not by a fixed direction.

## Design Contract Details (surface-by-surface)

### 1. Global tag filter (TAG-04)

- **Control:** a single `Select` (shadcn), visually identical to `DashboardFilters`'s preset `SelectTrigger` (`w-[170px]`, `aria-label="Filtro tag"`).
- **Placement:** in the existing filter row, alongside the tab-specific controls — on Overview: next to `OverviewHeader`'s year select; on Categorie: inside `DashboardFilters`'s flex row, after the preset select.
- **Value propagation:** a `?tag=<tagId>` URL param, carried across tab switches by extending `buildDashboardTabHref` the same way `preset`/`type`/`sort` already are (read-only integration note for the planner — not a new visual pattern).
- **Selection model:** single-select (not multi) — mirrors the existing month/year and preset selects being single-value; matches the design note's motivating case ("quanto ho speso per la vacanza a Sharm 2026" — one trip context at a time).
- **Options:** "Tutti i tag" (default/clear) + every tag (active and archived, archived suffixed with the existing "Archiviato" `Badge variant="secondary" text-[10px]`) — never a separate/hidden archived group.
- **Reconciliation contract (data, referenced not owned by this spec):** when a tag filter is active, every widget on the current tab narrows to transactions carrying that tag; the sums shown must still reconcile (TAG-04 success criterion) — a DAL/service concern, not visual, but the empty-state row above covers its zero-match UI outcome.

### 2. Tag section (TAG-05)

- **Location:** a new third tab in `DashboardTabNav`, "Tag", after "Categorie" (`/dashboard/tags`), reusing the exact tab-link visual contract (`border-b-2 border-primary text-primary` when active, `text-muted-foreground hover:text-foreground` otherwise). **Flagged as a default assumption** — no explicit route was locked upstream; the planner may relocate under a different path if a stronger precedent surfaces, but the tab-nav pattern itself should not be reinvented.
- **Card shape:** reuses `CategoryRankingList`'s card shell (`rounded-xl border bg-card p-4 shadow-sm transition-colors hover:border-primary/50`) **minus** the percentage bar and sparkline/deviation badge (no baseline/whole concept applies to a tag) **plus** the "Archiviato" badge when applicable.
- **Card content, top to bottom:**
  1. Tag name as a `<Link>` (truncate + `title=`), styled like `CategoryRankingList`'s title link (`text-sm font-semibold`, `hover:underline`, `focus-visible:ring-2`) — optional click-through target per the "unresolved" row above.
  2. "Archiviato" `Badge variant="secondary" text-[10px]` inline next to the name, only if archived (never removes the card, never disables anything — D-04 carried over).
  3. Per-tag total: `font-mono text-sm font-semibold tabular-nums`, sign-colored (`--total-in`/`--total-out`) per the Color section above.
  4. Caption: `"{count} movimenti · {minDate}–{maxDate}"`, `text-xs text-muted-foreground`.
  5. "Archivia" button (destructive, small) in the card's action area, opening the existing `ArchiveTagDialog` unchanged.
- **Sort:** by absolute total descending (default) — no percentage-of-whole column exists, unlike Categorie's ranking, because TAG-05 explicitly forbids a sum-to-100% framing.
- **No pagination/cap:** all tags render (this is a management-adjacent view, not a top-N insight surface).

### 3. Dashboard savings/deviations row click-through (NAV-01)

- **Target surface:** `OverviewMoversPanel`'s three category-keyed columns ("Variazioni di entrate", "Dove hai risparmiato", "Dove hai speso di più") — the surface with an explicit `selectedMonth` state (`OverviewMoversSection`), matching NAV-01's literal "with a month selected" framing. The Categorie/deviation-ranking page uses a `preset` window, not a single selected month, and is out of NAV-01's scope.
- **Interaction:** each `<li>` row in `MoverList` becomes a `<Link>` (currently a plain `<li>`) wrapping the existing row content, no visual restructuring — add `hover:bg-muted/50 focus-visible:ring-2 focus-visible:ring-ring rounded-sm` to the existing `odd:bg-muted/30 rounded-sm px-1` row classes so the affordance reads as clickable without changing row height/spacing.
- **Href construction:** `/transactions?months={year}-{selectedMonth+1 padded}&category={m.categoryId}` for the three category-keyed columns; rows with a null `categoryId` (should not occur in these three columns — only the Accantonamenti/allocation column groups by `natureCode`) render as plain (non-linked) text, unchanged.
- **Accantonamenti (allocation) column:** explicitly OUT of NAV-01's literal scope (grouped by `natureCode`, not `categoryId`, and the requirement names "savings/deviations" — i.e., the OUT/IN columns — not allocation). Left as a plain, non-clickable list, unchanged from its current implementation.
- **Empty/loading states:** unchanged — the existing "Nessuna variazione…" / `Loader2` spinner states already cover the empty/loading rails for this surface; no new copy needed since a non-populated column has no rows to make clickable.
