    isNotSecondary` from `@/lib/dal/transaction-pairs-sql`; `DASHBOARD_TOTAL_EXPENSE_STATUSES`
    from `@/lib/dal/dashboard`. Define `export type TagTotalItem = { tagId: number; name: string;
    archived: boolean; count: number; minDate: string | null; maxDate: string | null; total:
    string }` (signed total — sign-colored per the UI-SPEC, a tag can net positive or negative).
    Write `export async function getTagTotals(userId: string): Promise<TagTotalItem[]>` rooted at
    `FROM tag` (NEVER `FROM transaction` — the join direction is inverted vs every other dashboard
    aggregate in this codebase, per the Anti-Pattern warning in 68-RESEARCH.md) with LEFT JOINs:
    `transactionTag` (on `tagId`), `transactionTable` (on `transactionId`), `expense`, `subCategory`,
    `category`, `userSubcategoryOverride` (scoped additionally by `userId`), `nature` (via the
    `COALESCE(override.natureId, subCategory.natureId)` correlated join, matching
    `getCategoryRanking`'s exact shape), `direction`. The outer `.where()` is ONLY
    `eq(tag.userId, userId)` — critically, do NOT put the dashboard exclusions
    (status/transfer/pair-netting) in the outer WHERE, because that would silently drop a
    zero-transaction tag's row entirely via the LEFT JOINs. Instead apply the exclusions INSIDE
    each aggregate expression using a SQL `FILTER (WHERE ...)` clause: `count` becomes
    `count(distinct transactionTable.id) FILTER (WHERE <exclusion predicate>)`; `minDate`/`maxDate`
    similarly `MIN/MAX(transactionTable.occurredAt) FILTER (WHERE <exclusion predicate>)`; `total`
    becomes `coalesce(sum(effectiveAmount()) FILTER (WHERE <exclusion predicate>), 0)::text`. The
    shared `<exclusion predicate>` is: `expense.status = ANY` the `DASHBOARD_TOTAL_EXPENSE_STATUSES`
    array AND `direction.code != 'transfer'` AND the `isNotSecondary()` NOT-EXISTS condition —
    composed as one reusable local `sql` fragment so it is written once, not four times.
    `.groupBy(tag.id, tag.name, tag.archived)`. Do NOT `ORDER BY` in SQL — instead write a small
    pure `buildTagTotalsData(rows): TagTotalItem[]` function (exported, unit-testable without a
    DB, mirroring this codebase's `buildCategoryRankingData`/`buildOverviewData` query-vs-shaping
    split) that sorts by `toDecimal(total).abs()` descending in JS and returns the typed array;
    `getTagTotals` calls the query then delegates to `buildTagTotalsData`.
  </action>
