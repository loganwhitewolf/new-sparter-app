<objective>
Thread an optional trailing `tagId` through the three `lib/dal/overview.ts` exports that power
the Overview tab (`getOverview`, `getOverviewChart`, `getMonthOverMonthCategoryChanges`), add
`category.slug` to the movers-row data (fixing NAV-01's slug-vs-id mismatch, Pitfall 2), and fix
`fetchMovers`'s dropped-tagId regression on month-reselection (Pitfall 4).

Purpose: `getOverview` calls `getOverviewAmountTotals`/`getUncategorizedCount` (Plan 68-02)
directly, so this plan hard-depends on that signature change already existing.
`getMonthOverMonthCategoryChanges`'s `category.slug` addition is the ONE real gotcha this whole
phase's research flagged: the transactions filter contract matches `category` by slug, not id —
skipping this fix means NAV-01's click-through silently filters nothing.

Output: Three threaded exports in `overview.ts`, `MonthOverMonthChange.categorySlug`, and a fixed
`fetchMovers` action, ready for Plan 68-06 (page wiring) and Plan 68-07 (movers-row Link) to
consume.
</objective>
