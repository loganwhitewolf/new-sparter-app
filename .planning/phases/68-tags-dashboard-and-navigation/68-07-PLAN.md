<objective>
Wrap each `<li>` row in `OverviewMoversPanel`'s `MoverList` (the "Variazioni di entrate", "Dove hai
risparmiato", "Dove hai speso di più" columns) in a `<Link>` to
`/transactions?months={year}-{month}&category={categorySlug}`, using the `categorySlug` field
Plan 68-03 added to `MonthOverMonthChange` — never the numeric `categoryId` (Pitfall 2, the one
real gotcha this phase's research flagged: the transactions filter contract matches `category` by
slug, not id).

Purpose: This is the entirety of NAV-01. It requires zero new backend work — Plan 68-03 already
added `categorySlug`, and `parseTransactionFilters`/`getTransactions` already validate and apply
`category` by slug (pre-existing, unchanged). This plan is pure `<Link>` wiring in one existing
client component.

Output: Three of the four movers-panel columns become clickable, navigating to the correctly
pre-filtered transactions list; the fourth (Accantonamenti) stays unchanged, explicitly out of
scope.
</objective>
