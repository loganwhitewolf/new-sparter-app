<objective>
Audit the `vacanze` (Viaggi) category so it holds only intrinsically-travel spend: deactivate the
two overlapping subcategories (`attivita-e-intrattenimento`, `cibo-e-bevande`) via an additive
seed-extras step that first resets any currently-linked expenses to "da categorizzare", then flips
`isActive=false` on both subcategory rows; and add a new travel-only `trasporto` regex pattern so
future imports of flight/ferry/rental-car spend land in `vacanze` while daily-commute spend does not.

Purpose: TAG-06 is fully independent of the tag schema/DAL work in this phase (it touches only the
existing category taxonomy and regex machinery) and gives trip-tagging (Phase 68's per-tag view)
a clean categorization substrate underneath it — restaurant-on-vacation spend stops being hidden
inside Viaggi and instead surfaces truthfully under Ristorazione once the user re-categorizes it.

Output: one new `scripts/seed-extras.ts` STEP, one new `scripts/seed-patterns-data.ts` pattern
entry, and both existing test files extended to cover the new behavior.
</objective>
