<objective>
Build the `/settings/tags` curated tag list (create/edit/archive, D-01..D-04) and wire the
create-time suggestion trigger (D-08a): after creating a date-ranged tag, fetch matching
transactions and, if any exist, offer a pre-checked confirm dialog before closing out.

Purpose: this is the TAG-01 management surface every other plan's tag list ultimately points
back to, and the first of D-08's two suggestion triggers. It depends on Plan 67-03 (CRUD actions)
and Plan 67-05 (suggestion compute + confirm), so it ships after both are stable.

the `/settings/tags` page, and a new SettingsHub card — all covered by static-render + pure-helper
unit tests (this repo has no jsdom; multi-step dialog logic is extracted as pure/async functions
and tested directly, per the established `tests/merge-expenses-dialog.test.tsx` precedent).
</objective>
