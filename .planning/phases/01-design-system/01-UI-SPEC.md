## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge inner padding |
| sm | 8px | Compact element spacing, nav item gaps |
| md | 16px | Default element spacing, card padding, form field gap |
| lg | 24px | Section padding, sidebar padding, topbar height content |
| xl | 32px | Layout gaps, page content padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets (mobile bottom nav icons): minimum 44px hit area per WCAG 2.5.5, even if visual element is smaller
- Sidebar width (desktop): 240px (not on scale — structural value)
- Topbar height: 56px (closest multiple of 8 above standard 48px — accommodates logo + avatar with comfortable padding)

### Financial Numbers (Geist Mono)

All monetary amounts, KPI values (totalIn, totalOut, balance, savingsRate), count badges, and numbers in tables/lists MUST use Geist Mono at the same size/weight as their surrounding context. This ensures tabular alignment and visual distinction between numeric and text content.

### CSS Variables

shadcn/ui New York preset with slate base and emerald primary maps to:
- `--background`: white (dominant surface)
- `--foreground`: slate-900
- `--card`: white / slate-50
- `--card-foreground`: slate-900
- `--primary`: emerald-600 (#059669)
- `--primary-foreground`: white
- `--secondary`: slate-100
- `--secondary-foreground`: slate-900
- `--muted`: slate-100
- `--muted-foreground`: slate-500

- `--accent-foreground`: slate-900
- `--destructive`: red-500 (#ef4444)

- `--ring`: emerald-600

*Hover and focus variants (e.g. emerald-700 for primary hover) are derived automatically from the shadcn/ui Tailwind plugin — Claude's discretion per CONTEXT.md.*

### Accent Reserved For (explicit — not "all interactive elements")

1. Primary CTA buttons (Button with `variant="default"`)
2. Active sidebar item indicator (left border or background highlight on current route)
3. Positive KPI values: totalIn, savingsRate when positive
4. Uncategorized badge (Badge on "Categorie" nav item and Dashboard KPI section) — this is a call-to-action, not an error indicator

**Source:** CONTEXT.md D-06, D-07, specifics note on badge

## Layout Shell Contract

### (auth) Route Group — Public Pages

- No sidebar, no topbar
- Centered logo (Sparter wordmark) above the form
- Form centered vertically and horizontally on the page
- Background: dominant surface (white / slate-50)
- Used by: `/login` (Phase 1 stub), future `/register`

**Source:** CONTEXT.md D-05, D-11

### (app) Route Group — Authenticated Shell

**Sidebar (desktop, >= 768px):**
- Width: 240px fixed
- Background: slate-100 (secondary surface)
- Nav items (top): Dashboard, Spese, Import, Categorie
- Nav items (bottom, with Separator above): Impostazioni
- Active item: accent emerald-600 left border + emerald-50 background
- "Categorie" item carries a Badge (emerald-600) showing uncategorized expense count

**Topbar (desktop and mobile):**
- Height: 56px
- Logo: Sparter wordmark, left-aligned
- Avatar: right-aligned, opens DropdownMenu with items: Profilo, Logout
- No page title in topbar — title appears in page body

**Bottom Navigation (mobile, < 768px):**
- Position: fixed bottom, full width
- Background: white with top border slate-200
- Items (4): Dashboard, Spese, Import, Categorie
- Impostazioni: NOT in bottom nav — accessible via avatar dropdown only
- Touch targets: minimum 44px height per WCAG 2.5.5

**Source:** CONTEXT.md D-01, D-02, D-03, D-04, D-05

**Breakpoint:** `md` (768px) — sidebar to bottom-nav transition. Claude's discretion per CONTEXT.md.

## Implementation Notes for Executor

1. **Font loading:** Use `next/font/google` for both Geist and Geist Mono. Apply as CSS variables to `<html>` in `app/layout.tsx`. No external font requests at runtime.

2. **Geist Mono application:** Apply via a utility class (e.g. `font-mono`) to: all `amount` fields, KPI value elements, Badge count displays, table number cells.

3. **shadcn init command:** `npx shadcn@latest init` with New York style, slate base color, emerald primary, 0.5rem radius. Confirm `components.json` exists before proceeding.

4. **CSS variables:** shadcn/ui injects variables into `globals.css` — do not override manually. Extend via Tailwind config for project-specific tokens (e.g. `--color-totalIn`, `--color-totalOut`).

5. **Route groups:** Create `app/(auth)/layout.tsx` (minimal, centered) and `app/(app)/layout.tsx` (sidebar + topbar shell) in Phase 1. Page stubs: `app/(auth)/login/page.tsx` and `app/(app)/dashboard/page.tsx`.

6. **Directory structure:** Create empty placeholder files for `lib/db/`, `lib/dal/`, `lib/services/`, `lib/actions/` per CLAUDE.md. Add `import 'server-only'` to `lib/db/index.ts`.

7. **Border radius exception:** New York preset default is `0.5rem` (8px). CONTEXT.md D-08 mentions 6px button / 8px card — the preset value `0.5rem` (8px) takes precedence as it is the shadcn configuration value. Button radius may render at 6px via New York style internal defaults; do not override.
