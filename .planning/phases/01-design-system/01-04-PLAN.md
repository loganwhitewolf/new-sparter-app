**Step 1: Create lib/db/index.ts — server-only guard + typed Drizzle stub**

```typescript
// lib/db/index.ts
// CRITICAL: import 'server-only' prevents this file from being imported in client components.
// Any attempt to import lib/db from a client component will throw at build time.
import 'server-only'

// Drizzle ORM PostgreSQL client
// Phase 1: stub only — no live DB connection until Phase 2 wires DATABASE_URL
// Do NOT import this file from any UI component or page in Phase 1
//
// Phase 2 will replace this stub with:
//   import { drizzle } from 'drizzle-orm/node-postgres'
//   import { Pool } from 'pg'
//   import * as schema from './schema'
//   const pool = new Pool({ connectionString: process.env.DATABASE_URL! })
//   export const db = drizzle(pool, { schema })

export const db = null as unknown as import('drizzle-orm/node-postgres').NodePgDatabase

// DbOrTx: used by all DAL helpers and importFile() in Phase 5+
// Accepts either the db instance OR a transaction callback argument
// Ensures all operations inside importFile() share the same transaction
export type DbOrTx = typeof db
```

| client components → lib/db | server-only guard prevents db from reaching browser bundle |
| shadcn CLI → node_modules | Radix UI primitives installed via npm |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | lib/db/index.ts | mitigate | `import 'server-only'` is the mitigation — it throws a build error if any client component tries to import db. This is enforced at build time, not runtime. |
| T-04-02 | Tampering | shadcn add (supply chain) | mitigate | Use `npx shadcn@latest add` — official CLI. Components are local TypeScript files after install; inspect them if any look unexpected. |
| T-04-03 | Tampering | Radix UI primitives (supply chain) | accept | Radix is maintained by WorkOS; widely used; no known supply chain issues. Monitor with `npm audit`. |
| T-04-04 | Elevation of Privilege | DbOrTx type in Phase 5+ | mitigate | DbOrTx is a type alias only — no runtime privilege. The actual db object is null in Phase 1. Phase 2+ must validate DATABASE_URL before initializing. |
</threat_model>
