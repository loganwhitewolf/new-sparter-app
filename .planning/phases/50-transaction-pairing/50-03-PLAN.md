<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → server action | Untrusted `transactionId`/`counterpartId` cross here; Zod validates shape, verifySession establishes identity, service validates ownership |
| server action → service → DB | The pair table has no userId (D-01); the service is the sole authorization gate before insert/delete |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-01 | Information Disclosure / Tampering (IDOR) | `createPair` / `deletePairByTransactionId` / `getEligibleCounterparts` | mitigate | createPair loads BOTH transaction rows and throws if either userId !== session userId before insert; deletePairByTransactionId checks the transaction's userId before delete; getEligibleCounterparts is `verifySession`-scoped with `eq(transaction.userId, userId)` so counterparts outside the user's data are never enumerated. Acceptance: service ownership tests GREEN + `eq(transaction.userId` present in picker DAL. |
| T-50-02 | Tampering (double-link) | unique constraints + picker pre-filter | mitigate | DB unique constraints (Plan 02) reject a second pair; the picker's `NOT EXISTS` already-paired filter (D-14) prevents the UX from ever offering an already-paired counterpart; createPair surfaces the constraint violation as a thrown Error. Acceptance: double-link service test GREEN + NOT EXISTS in picker DAL. |
| T-50-03 | Input Validation | `CreatePairSchema` / `DeletePairSchema` | mitigate | Zod validates IDs as non-empty strings at the action boundary before any DB access (ASVS V5). Acceptance: parse-failure returns `{ error }` without calling the service. |
| T-50-04 | Spoofing | action identity | mitigate | `verifySession()` runs in every action after parse and before the service call (ASVS V4). Acceptance: both actions call verifySession. |
| T-50-SC | Tampering | npm/pip/cargo installs | accept | No package installs (50-RESEARCH.md §Package Legitimacy Audit: not applicable). |
</threat_model>
