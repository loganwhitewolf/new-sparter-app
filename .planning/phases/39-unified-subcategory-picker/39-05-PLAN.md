<objective>
Rework the two pattern-authoring surfaces per D-08/D-09/R-UP-07 and ADR 0008. The `create-pattern-dialog` becomes: a regex input + a description input + a "Categorizza" button that opens the unified `SubcategoryPicker`. The explicit Categoria/Sottocategoria `Select` pair, the "Segno importo" control, and the "Confidenza" input are all removed. The `suggestion-promote-form` likewise drops its cascading Selects in favor of the picker (keeping its pre-filled regex). `amountSign` is no longer a user field: it is derived **server-side** from the chosen subcategory's category type (`out -> negative`, `in -> positive`, `transfer`/`system -> any`); `confidence` is hardcoded to `1` for both create and promote. The `amountSignMatches` matching logic stays untouched.

Purpose: Align pattern authoring with the unified picker and ADR 0008 (amountSign is a property of the subcategory's category, not a separate user choice).
Output: Both pattern forms use the picker; both actions derive amountSign server-side and write confidence=1; a unit test pins the derivation table.
</objective>
