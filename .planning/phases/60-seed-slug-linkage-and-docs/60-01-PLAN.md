<objective>
Remove explicit hardcoded integer ids from seeded platform entries and switch the seed-time linkage between import format versions and their platform from a numeric FK to a runtime slug resolution ŌĆö closing the Trade Republic id-8 collision (PLAT-05).

Purpose: When a user platform already holds whatever serial integer the seeded Trade Republic entry was hardcoded to, the old seed silently skipped the TR format version (onConflictDoNothing fired on the wrong constraint). After this change, the platform insert is id-free (slug-keyed), the format version insert derives its FK from a fresh post-insert query, and the collision cannot recur regardless of which integer the serial assigns.

Output:
- scripts/seed-data.ts ŌĆö no numeric id on any platform entry; each format version entry carries a platformSlug string
- scripts/seed.ts ŌĆö slug-to-integer-id resolution between the platform insert and the format version insert
</objective>
