<objective>
Wire the Phase 52 correctness gates into the Phase 51 `discoverRegexCandidates` service. Four LOCKED behaviors, all additive on the existing service body (52-CONTEXT decisions):

1. **RDISC-01 / RDISC-02 routing** — after clustering, split the suggestions on `residualVariablePart`: non-empty → `candidates` (genuine prefix+variable regex families); empty → `singleCategorizationSuggestions` (identical-after-normalization groups, NO regex). The signal already exists (Phase 51 `residualVariablePart`); no new clustering algorithm.
2. **RDISC-03 Check 1** — drop any regex family whose generated regex an existing active pattern already covers, using `candidateCoveredByExistingPattern` (Plan 01). Distinct from the Phase 51 input-row coverage filter, which stays.
3. **RDISC-04 Check 2** — carry `descriptionHash` per detector row through the util, collect each suggestion's member hashes, call `getManuallyCategorizedHashes` (Plan 02), and drop ANY suggestion (regex or single) with a member hash in the manual set (any-member policy, platform-agnostic, against the user's ENTIRE manual history — LOCKED).
4. **Output shape** — extend `DiscoveryResult` with a second list `singleCategorizationSuggestions: SingleCategorizationSuggestion[]` (NOT a discriminated union). Downstream contract for Phase 54 (trigger) and Phase 55 SUMUI-02 (visual separation).

Purpose: This is the orchestration tier that delivers all four phase success criteria; the two DoD test cases live here.
Output: Extended service body, new `SingleCategorizationSuggestion` type, two-list `DiscoveryResult`, and the RDISC-01/02/03/04 service tests.
</objective>
